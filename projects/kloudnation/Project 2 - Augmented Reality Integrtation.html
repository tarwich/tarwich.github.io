Background

KloudNation is currently developing an iPhone and Android App. Currently the iPhone app is functional and has a UI. The iPhone app needs an Augmented Reality screen added to the already existing code base. When attempting to add an Augmented Reality screen, the project ceased to compile, and KloudNation would like assistance resolving this issue.

Objectives

Project Objectives

  • Resolve current compilation issues
  • Integrate Augmented Reality into the project
  • Enhance beacon tracking

Constraint Prioritization

Schedule
Schedule is of the utmost importance. Project must meet customer expectations in regards to schedule.
Scope
Scope is somewhat flexible. Scope may be adjusted if necessary in order to meet the schedule constraint, but should be as close to the original plan as possible.
Budget
Budget is the most flexible. Adjusting budget will be the primary method to adapting to project changes in order to meet schedule and scope constraints

Scope

In Scope

  • Resolve current compilation issues by adjusting settings or removing componentes that will no compile
  • Integrate the Vuforia framework into the existing project. If not possible, then substitute an Augmented Reality framework that will compile into the project
  • Add an Augmented Reality screen to the application
  • Make the Augmented Reality screen download data from the KloudNation server in order to get Tracking images and movies that will play when those tracking images are "seen" by the phone.
  • Make the phone auto-play the videos when in view
  • Change the beacon system to send beacon updates every time anything changes about the top three beacons
  • Change the beacon system to weed out irregular beacon data

Deliverables

  • KloudNation     All items in Equipment and Facility Needs
  • Samuel Dillow     All items In Scope

Schedule

This project should begin on Tuesday at 11:00 am. Samuel Dillow can consume as much time as needed in order to complete the items In Scope. The project must be completed NLT  Thursday, 23 Oct 2014

Risks

  • This project has a poorly defined scope. As such, the expectations are subject to change throughout. The potential impact of this risk is that the project could drastically increase in scope, and, therefore, schedule.

    Remediation: Establish a tighter scope, or set a budget limit to prevent scope creep.

Assumptions

  • The UI is in a functional state.
  • The environment in which the Augmented Reality will be utilized has been determined.
  • The radio beacons have been aquired by KloudNation and will not change at the time of using the application.
  • There are a finite number of targets, which have been determined by KloudNation.
  • The web server for delivering the Augmented Reality content is in a stable state.
